## char

Live transcription and audio tools

```
char [--base-url] [--api-key] [-m] [-l] [--record] [--no-color] [--base] [-v] [-q] [-h] [-V] <command>
```

**Global options:**

- `--base-url <BASE_URL>`
- `--api-key <API_KEY>`
- `-m, --model <MODEL>`
- `-l, --language <LANGUAGE>` (default: `en`)
- `--record`
- `--no-color`
- `--base <DIR>`
- `-v, --verbose` ‚Äî Increase logging verbosity
- `-q, --quiet` ‚Äî Decrease logging verbosity

**Options:**

- `-h, --help` ‚Äî Print help
- `-V, --version` ‚Äî Print version

**Subcommands:**

- `chat` ‚Äî Interactive chat with an LLM
- `listen` ‚Äî Start live transcription (TUI)
- `connect` ‚Äî Configure an STT or LLM provider
- `sessions` ‚Äî Browse past sessions
- `humans` ‚Äî Browse humans (contacts)
- `orgs` ‚Äî Browse organizations
- `status` ‚Äî Show configured providers and settings
- `auth` ‚Äî Authenticate with char.com
- `desktop` ‚Äî Open the desktop app or download page
- `bug` ‚Äî Report a bug on GitHub
- `hello` ‚Äî Open char.com
- `batch` ‚Äî Transcribe an audio file
- `models` ‚Äî Manage local models
- `completions` ‚Äî Generate shell completions

---

### char chat

Interactive chat with an LLM

```
char chat [--prompt] [--provider] [-h] [-V] <command>
```

**Options:**

- `--prompt <PROMPT>` ‚Äî Send a single prompt without entering the TUI (use `-` to read from stdin)
- `--provider <PROVIDER>`
- `-h, --help` ‚Äî Print help
- `-V, --version` ‚Äî Print version

**Subcommands:**

- `resume` ‚Äî Resume an existing chat session
